# DGA306 Day 4 — การวิเคราะห์ข้อมูลเปิดภาครัฐ

## ภาพรวมโครงการ

โครงการนี้เป็นส่วนหนึ่งของการอบรมหลักสูตร **DGA306: ทักษะด้านข้อมูลเพื่อการทำงานภาครัฐ (Data Skills for Government Officers)** จัดโดยสำนักงานพัฒนารัฐบาลดิจิทัล (องค์การมหาชน) หรือ DGA

วัตถุประสงค์หลักของโครงการคือการนำข้อมูลเปิดภาครัฐ (Open Government Data) จากแหล่งข้อมูลจริงของหน่วยงานรัฐมาวิเคราะห์ ทำความสะอาด สร้างแผนภาพแบบโต้ตอบได้ (Interactive Visualization) และทดลองใช้แบบจำลองการเรียนรู้ของเครื่อง (Machine Learning) เพื่อพยากรณ์แนวโน้มข้อมูล พร้อมนำเสนอผลลัพธ์ทั้งหมดผ่านแดชบอร์ดแบบโต้ตอบ (Interactive Dashboard) ที่เปิดดูได้ผ่านเว็บเบราว์เซอร์โดยไม่ต้องติดตั้งโปรแกรมเพิ่มเติม

## สิ่งที่จะได้เรียนรู้จากโครงการนี้

โครงการนี้ครอบคลุมทักษะพื้นฐานและกระบวนการทำงานกับข้อมูล ตั้งแต่ต้นจนจบ ดังนี้

### การจัดการข้อมูล (Data Management)
- การดาวน์โหลดข้อมูลจากแหล่งข้อมูลเปิดภาครัฐ (Open Government Data) อย่างเป็นระบบ
- การจัดการไฟล์ที่มีรูปแบบการเข้ารหัสอักขระแตกต่างกัน (เช่น UTF-8, CP874/TIS-620) และตัวคั่นข้อมูล (delimiter) ที่ไม่ตรงกัน
- การรวมข้อมูลหลายไฟล์หลายปีเข้าด้วยกัน โดยรักษาความสมบูรณ์ของข้อมูล

### การทำความสะอาดข้อมูล (Data Cleaning)
- การตรวจสอบและปรับชื่อคอลัมน์ให้เป็นมาตรฐานเดียวกันข้ามปีข้อมูล
- การจัดการค่าว่าง (Missing Values) และการแปลงประเภทข้อมูล
- การแปลงข้อมูลจากรูปแบบกว้าง (Wide Format) เป็นรูปแบบยาว (Long Format) เพื่อให้เหมาะกับการวิเคราะห์

### การแสดงผลข้อมูล (Data Visualization)
- การสร้างแผนภาพเส้น (Line Chart) แสดงแนวโน้มการจำหน่ายน้ำมันรายเดือน
- การสร้างแผนภาพพื้นที่แบบซ้อน (Stacked Area Chart) แสดงสัดส่วนประเภทน้ำมัน
- การสร้างแผนที่ความร้อน (Heatmap) แสดงรูปแบบตามฤดูกาล
- การสร้างแผนภาพแท่ง (Bar Chart) เปรียบเทียบข้อมูลคดีคุมประพฤติตามจังหวัดและฐานความผิด
- การสร้างแผนภาพกระจาย (Scatter Plot) จากการจัดกลุ่มข้อมูลด้วย K-Means Clustering

### การเรียนรู้ของเครื่อง (Machine Learning)
- การสร้างแบบจำลองพยากรณ์ (Predictive Model) ด้วย Linear Regression
- การเตรียมตัวแปรอิสระ (Features) จากข้อมูลอนุกรมเวลา (Time Series)
- การแบ่งข้อมูลสำหรับฝึกสอนและทดสอบ (Train/Test Split) แบบตามลำดับเวลา
- การประเมินความแม่นยำด้วยค่า RMSE และ R-squared
- การวิเคราะห์ข้อจำกัดของแบบจำลอง (เช่น ผลกระทบจากสถานการณ์ COVID-19)

### การนำเสนอข้อมูล (Data Presentation)
- การรวมแผนภาพจากหลายสมุดโน้ตเข้าเป็นแดชบอร์ดเดียว
- การเพิ่มบริบทและการเล่าเรื่องด้วยข้อมูล (Data Storytelling)
- การสร้างไฟล์ HTML แบบ standalone ที่เปิดดูได้ทันทีในเว็บเบราว์เซอร์

## รายละเอียดสมุดโน้ตและแดชบอร์ด

### สมุดโน้ตที่ 1 — แนวโน้มการจำหน่ายน้ำมันดีเซลหมุนเร็ว บี 7
**ไฟล์:** `notebooks/doeb_fuel_trend_viz.ipynb`

วิเคราะห์แนวโน้มปริมาณการจำหน่ายน้ำมันดีเซลหมุนเร็ว บี 7 ย้อนหลัง 8 ปี (พ.ศ. 2560–2567) ประกอบด้วยแผนภาพ 3 ชุด ได้แก่
- **แผนภาพเส้นหลายเส้น (Multi-line Chart):** เปรียบเทียบปริมาณจำหน่ายรายเดือนระหว่างน้ำมัน ADO B7 และ ADO
- **แผนภาพพื้นที่แบบซ้อน 100% (Stacked Area 100%):** แสดงสัดส่วนของน้ำมันแต่ละประเภทในแต่ละเดือน
- **แผนที่ความร้อนตามฤดูกาล (Seasonal Heatmap):** แสดงรูปแบบการจำหน่ายตามเดือนและปี

### สมุดโน้ตที่ 2 — พยากรณ์ความต้องการน้ำมัน บี 7 ด้วย Machine Learning
**ไฟล์:** `notebooks/doeb_fuel_forecast_ml.ipynb`

ทดลองใช้โมเดล Linear Regression เพื่อพยากรณ์ปริมาณความต้องการน้ำมัน บี 7 ในอนาคต ประกอบด้วย
- **แผนภาพค่าจริงเทียบกับค่าพยากรณ์ (Actual vs Predicted):** แสดงผลการทำนายของโมเดล
- **แผนภาพค่าความคลาดเคลื่อน (Residuals Chart):** วิเคราะห์ข้อผิดพลาดของโมเดล
- **ค่าวัดประสิทธิภาพ:** RMSE และ R-squared สำหรับชุดข้อมูลฝึกสอนและทดสอบ

### สมุดโน้ตที่ 3 — คดีคุมความประพฤติผู้กระทำผิดที่เป็นผู้ใหญ่
**ไฟล์:** `notebooks/probation_cases_viz.ipynb`

วิเคราะห์ข้อมูลคดีคุมความประพฤติปีงบประมาณ 2565–2566 ประกอบด้วย
- **แผนภาพแท่ง 10 จังหวัดสูงสุด (Top 10 Bar Chart):** จังหวัดที่มีคดีมากที่สุด
- **แผนภาพแท่งแบบซ้อน (Stacked Bar):** จำแนกตามฐานความผิด
- **แผนภาพเส้นรายเดือน (Monthly Line Chart):** แนวโน้มคดีตามเวลา
- **แผนภาพการจัดกลุ่ม (K-Means Clustering):** จัดกลุ่มจังหวัดตามรูปแบบคดี

ไฟล์ต้นแบบ (Starter Kit) ที่ได้รับจากหลักสูตร DGA306 ใช้เป็นแนวทางเริ่มต้นสำหรับผู้ที่ต้องการศึกษาตัวอย่างการดึงข้อมูล การทำความสะอาด และการสร้างแผนภาพเบื้องต้นก่อนเริ่มทำโครงการของตนเอง

## ข้อมูลสำคัญและข้อจำกัด

### ขอบเขตของข้อมูล
- **ข้อมูลน้ำมัน (DOEB):** ครอบคลุมเฉพาะผู้ค้าน้ำมันตามมาตรา 7 เท่านั้น ไม่รวมผู้ค้ารายเล็กหรือช่องทางจำหน่ายอื่น ดังนั้นตัวเลขที่แสดงจึงไม่ใช่ปริมาณการใช้น้ำมันทั้งหมดของประเทศ
- **ข้อมูลคุมประพฤติ (Probation):** ใช้ปีงบประมาณ (ตุลาคม – กันยายน) ไม่ใช่ปีปฏิทิน แกนเวลาในทุกแผนภาพจึงระบุเป็น "ปีงบประมาณ (ต.ค.–ก.ย.)" เพื่อความชัดเจน

### ข้อจำกัดของแบบจำลอง Machine Learning
- แบบจำลอง Linear Regression ที่ใช้เป็นแบบจำลองพื้นฐาน (Baseline Model) เพื่อการเรียนรู้ ไม่เหมาะสำหรับใช้ในการตัดสินใจเชิงนโยบาย
- ช่วงข้อมูลฝึกสอนครอบคลุมช่วง COVID-19 (พ.ศ. 2563) ซึ่งทำให้ค่าสัมประสิทธิ์แนวโน้ม (Trend Coefficient) ติดลบ ขัดแย้งกับการฟื้นตัวของความต้องการน้ำมันในช่วงทดสอบ ส่งผลให้ค่า R-squared ของชุดทดสอบติดลบ
- ใช้ข้อมูลรวมระดับประเทศ (Aggregate Data) จึงไม่สามารถจับรูปแบบเฉพาะพื้นที่ได้

### การเข้ารหัสอักขระ (Encoding)
- ข้อมูล DOEB ปี 2560–2566 ใช้ UTF-8 ส่วนปี 2567 ใช้ CP874
- ข้อมูลคุมประพฤติ (CSV ต้นฉบับ) ใช้ CP874 โดยไฟล์ปี 2566 ใช้ Tab เป็นตัวคั่นแทน Comma
- ไฟล์ที่ผ่านการประมวลผลแล้วทั้งหมด (ในโฟลเดอร์ `data/processed/`) ใช้ UTF-8 with BOM (utf-8-sig)
